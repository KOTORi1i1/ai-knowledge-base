# ⚡ Claude Code Skills

> Claude Code 内置的斜杠命令（Slash Commands）。以 `/` 开头，每个命令赋予 Claude 特定的专业能力。

---

## 什么是 Claude Code Skills？

Claude Code 是 Anthropic 官方的命令行 AI 编程助手。Skills 是它的扩展命令系统，分为两类：

| 类型 | 说明 | 示例 |
|------|------|------|
| **内置命令** | Claude Code 自带的核心功能 | `/review`, `/compact`, `/init` |
| **自定义 Skills** | 用户或社区创建的扩展 | `/deep-research`, `/pdf-reader` |

---

## 📋 内置命令详解

### `/review` — 代码审查

<div class="skill-detail">

| 属性 | 内容 |
|------|------|
| **难度** | 🟢 新手友好 |
| **用途** | 自动审查代码，发现潜在 bug、安全漏洞、代码异味、性能问题 |
| **适用场景** | 提交 PR 前自查 / 审查团队代码 / 学习最佳实践 |

**使用方法：**

```bash
# 审查当前所有改动
/review

# 审查特定文件
/review src/app.ts

# 审查某个 PR
/review https://github.com/user/repo/pull/123
```

**审查维度：**
- 🐛 **Bug 检测**：空指针、边界条件、竞态条件
- 🔒 **安全检查**：注入风险、敏感信息泄露、权限问题
- ⚡ **性能优化**：无效循环、重复查询、内存浪费
- 📐 **代码风格**：命名规范、函数长度、注释质量

**实测评价：** ✅ 能发现大多数常见问题，尤其是空值检查和异常处理；对于复杂业务逻辑需要人工复核。

</div>

### `/compact` — 上下文压缩

<div class="skill-detail">

| 属性 | 内容 |
|------|------|
| **难度** | 🟢 新手友好 |
| **用途** | 压缩长对话的上下文，让 Claude 自动总结历史并继续工作 |
| **适用场景** | 对话很长导致响应变慢 / 需要切换话题前 / 省钱省 token |

**使用方法：**

```bash
/compact
```

不需要任何参数。Claude 会自动总结前面的对话，保留关键信息，然后继续。

**实测评价：** ✅ 对话变长后必用，相当于"存档并开新档"。注意：压缩后 Claude 会丢失一些细节，遇到复杂任务时建议在压缩前手动记录关键信息。

</div>

### `/init` — 项目初始化

<div class="skill-detail">

| 属性 | 内容 |
|------|------|
| **难度** | 🟢 新手友好 |
| **用途** | 一键分析项目结构，自动生成 CLAUDE.md 配置文件 |
| **适用场景** | 新项目启动 / 让 Claude 快速理解陌生代码库 |

**使用方法：**

```bash
# 在项目根目录执行
cd my-project
/init
```

Claude 会扫描项目文件结构、读取关键配置文件、分析依赖关系，然后生成一份 `CLAUDE.md`。之后每次在这个项目中工作，Claude 都会自动读取这份文件来理解项目上下文。

**生成的 CLAUDE.md 包含：**
- 项目简介和技术栈
- 目录结构说明
- 构建/运行/测试命令
- 编码规范和约定

**实测评价：** ✅ 非常好用的功能。生成的 CLAUDE.md 质量很高，可以手动微调后作为项目的"AI 说明书"长期使用。

</div>

### `/memory` — 记忆管理

<div class="skill-detail">

| 属性 | 内容 |
|------|------|
| **难度** | 🟢 新手友好 |
| **用途** | 管理 Claude Code 的持久化记忆，让 Claude 记住你的偏好和习惯 |
| **适用场景** | 告诉 Claude 你的编码风格 / 记住常用配置 / 跨项目共享偏好 |

**使用方法：**

```bash
# 查看所有记忆
/memory

# Claude 会在对话中自动帮你添加记忆
# 比如你说 "我喜欢用 TypeScript 而不是 JavaScript"
# Claude 会自动创建一条记忆
```

**记忆类型：**
- **用户信息**：你的角色、技术栈偏好
- **反馈记录**：你之前的修正意见，避免重复犯错
- **项目信息**：项目目标、架构约束
- **参考资料**：常用文档链接、API 地址

**实测评价：** ✅ 用久了效果明显——Claude 会越来越"懂你"。建议定期清理过时的记忆。

</div>

**实测评价：** ✅ 一次性配置，之后 Claude 就能在你的终端里直接操作了。Windows 用户注意：建议配合 Git Bash 或 WSL 使用。

## 🎯 使用建议

### 新手推荐学习顺序

```
第 1 天：/help、/init、/compact
第 2 天：/review、/memory
第 3 天：/config、/terminal-setup
第 4 天：/cost、/doctor
第 5 天：/agents、/workflows
```

### 常用工作流配合

```bash
# 场景一：接手一个新项目
/init              # 第 1 步：生成项目配置
/review            # 第 2 步：审查现有代码质量
/memory            # 第 3 步：记录项目偏好

# 场景二：日常开发
/review src/app.ts  # 写完代码后自查
/compact            # 对话变长后压缩
/cost               # 查看用量
```

## 📚 自定义 Skills

除了内置命令，Claude Code 还支持用户自定义 Skills。自定义 Skill 是一个 Markdown 文件，定义了特定的专业能力和行为模式。

**创建自定义 Skill：**

1. 在 `.claude/skills/` 目录下创建 `.md` 文件
2. 在文件中用 YAML frontmatter 定义 Skill 名称和描述
3. 用 Markdown 编写 Skill 的指令和行为规范

之后在对话中用 `/你的skill名` 即可调用。社区常用的自定义 Skills 包括：

- **PDF 阅读器**：让 Claude 能读取和分析 PDF 文件
- **代码生成器**：按模板生成标准代码文件
- **数据可视化**：自动生成图表和报告
- **API 文档查询**：快速查找 API 用法

> 💡 **提示**：发现好用的自定义 Skill？欢迎通过 GitHub Issue 推荐给我们！
